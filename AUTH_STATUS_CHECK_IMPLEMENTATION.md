# Реализация проверки статуса во всех API ответах

## Описание
Добавлена полная проверка атрибута `status` во всех ответах от API. Если `status !== 0`, то токен удаляется из cookies и пользователь перенаправляется на страницу авторизации.

## Файлы, которые были изменены

### 1. `src/api/axios.js`
- Создан интерцептор ответов (response interceptor)
- Проверка наличия `status` в ответе
- При `status !== 0`:
  - Удаление токена из cookies (`Cookies.remove('token')`)
  - Удаление email из cookies (`Cookies.remove('user')`)
  - Перенаправление на `/login` (если не на странице логина/регистрации)
- Обработка ошибок HTTP 401 и 403
- Экспортированы функции:
  - `setNavigate(navigate)` - для установки функции навигации
  - `setCurrentPath(path)` - для отслеживания текущей страницы

### 2. `src/App.js`
- Добавлен новый компонент `NavigationInitializer`
- Компонент оборачивает приложение и инициализирует:
  - Функцию навигации для интерцептора
  - Текущий путь для предотвращения перенаправления на /login со страницы логина

## Как это работает

1. **При загрузке приложения:**
   - Компонент `NavigationInitializer` уже имеет доступ к `useNavigate()` и `useLocation()`
   - Эти функции передаются в интерцептор axios

2. **При каждом API запросе:**
   - Интерцептор проверяет ответ
   - Если `response.data.status !== 0`:
     - Удаляет токен и email из cookies
     - Перенаправляет пользователя на страницу логина
   - Если есть ошибка 401 или 403:
     - Выполняет те же действия

3. **Исключения:**
   - На страницах `/login` и `/registration` перенаправление не происходит

## Преимущества

✅ **Глобальная защита** - Проверка работает на всех страницах и всех API запросах
✅ **Автоматическое удаление токена** - При невалидном токене он сразу удаляется
✅ **Бесперебойная работа** - Пользователь не застревает с невалидным токеном
✅ **Отсутствие дублирования** - Не нужно добавлять проверку на каждой странице отдельно

## Тестирование

Для тестирования:
1. Запустите приложение: `npm start`
2. Авторизуйтесь
3. Вручную измените или удалите токен в cookies (F12 → Application → Cookies)
4. Сделайте любой API запрос (например, перейдите на другую страницу)
5. Приложение должно перенаправиться на страницу логина и удалить cookies

## Заметки

- Локальные проверки `status` на странице логина/регистрации остаются без изменений
- Интерцептор работает прозрачно для существующего кода
- Все страницы получают автоматическую защиту без изменений
