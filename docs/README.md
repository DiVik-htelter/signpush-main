# 📚 Документация проекта SignPush

## Обзор проекта

**SignPush** — веб-приложение на React для управления электронными документами с функционалом цифрового подписания PDF-файлов.

### Основной функционал
- ✅ Аутентификация пользователей (email/password)
- ✅ Загрузка и просмотр PDF-документов
- ✅ Создание рукописных цифровых подписей
- ✅ Размещение подписей на документах
- ✅ Фильтрация документов по статусу
- ✅ Вычисление SHA-256 хеша для верификации
- ✅ Пагинация списков документов

### Технологический стек
- **Frontend:** React 18.2.0, React Router v6
- **UI Framework:** Bootstrap 5.3.2 + React-Bootstrap
- **PDF Engine:** PDF.js 3.10.111
- **HTTP Client:** Axios 1.5.1
- **State Management:** React Context API + Cookies
- **Build Tool:** Create React App (react-scripts 5.0.1)

### API Backend
- **Base URL:** https://test.signpush.ru/api/v4.1
- **Endpoints:**
  - `PATCH /user/s/auth/password` - Аутентификация
  - `GET /paper/document` - Получение списка документов

## ⚠️ Известные проблемы

### Критические
- 🚨 Hardcoded API ключ в коде (security risk)
- 🚨 Авторизация при ошибке сети в Login component
- 🚨 Опечатка в RequireAuth (`const { location } = useLocation()`)

### Средние
- ⚠️ Опечатка в axios.js (`baseUrl` вместо `baseURL`)
- ⚠️ Утечка памяти в PdfReader (event listeners не удаляются)
- ⚠️ Cookies вместо localStorage для токенов

### Низкие
- ℹ️ Дублирование PDF.js дистрибутивов (3MB+)
- ℹ️ Неиспользуемые переменные и функции
- ℹ️ Отсутствие обработки ошибок

Подробнее см. [SECURITY.md](./SECURITY.md)

## 📈 Производительность

### Сильные стороны
- ✅ Code splitting (динамический import PDF.js)
- ✅ Web Workers для PDF рендеринга
- ✅ React 18 с автоматическим batching

- ⚠️ Дублирование PDF.js версий

## 🔒 Безопасность

### Текущие риски
1. **API ключ в коде** — виден в bundle.js
2. **XSS уязвимость** — токены в cookies без HttpOnly
3. **Bypass аутентификации** — ошибка в Login catch блоке
4. **CORS** — нет проверки origin
5. **Слабая валидация** — только HTML5 validation

Подробнее см. [SECURITY.md](./SECURITY.md)
